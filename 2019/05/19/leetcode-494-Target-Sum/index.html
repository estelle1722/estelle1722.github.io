动态规划

最先想到的是使用dp[i][j]表示数组前\(i\)个元素组成目标元素\(j\)的方案数，这样动态转移方程如下：

\[ dp[i][j]=dp[i-1][j+nums[i]] + dp[i-1][j-nums[i]] \]

理解起来也非常容易，对于第\(i\)个元素，要么置为正，则此时需要用前\(i-1\)个元素组成\(j-nums[i]\)；否则，置为负，此时需要用前\(i-i\)个元素组成\(j+nums[i]\)；但是有个问题就是，这种方法要求目标数字不为负数，但是题目并没有这样保证；因此，是不能直接这样做的；接着我想到了既然不能用数组表示，那就用哈希表呗！我们用元组do[(i,target)]表示用前\(i\)个数字组成数字target的方案数，和上面动态转移方程是一样的，

\[ dp[(pos, target)] = dp[(pos + 1, target - nums[pos])] + dp[(pos + 1, target + nums[pos])] \]

最后结合数学推导，介绍一种效率更高的算法，因为上面算法就本质来说，还是属于dfs，效率较低；我们设被置为正数的元素组成的子集为\(P\)，负数组成的子集为\(Q\)，目标数字为\(T\)；依题意有：

\[ sum(P)-sum(Q)=T \\ sum(P)+sum(Q)+sum(P)-sum(Q)=T+sum(P)+sum(Q) \\ 2sum(Q)=T + sum(nums)\\ sum(Q)=\frac{T+sum(nums)}{2} \]

根据以上推导，我们就可以将该问题转化为求数组中元素和为\(\frac{T+sum(nums)}{2}\)的子集个数。当然，上式还需要满足以下必要条件：

  1. \(T+sum(nums)\ge 0\)
  2. \((T+sum(nums))\%2=0\)

可以理解为，我们将其转化为原问题的同解问题（有点像对偶问题的感觉）。我们可以使用动态规划来解决新问题，这里，我们可以使用dp[i][j]表示用前\(i\)个数字组成目标数字\(j\)的方案种数；这样动态转移方程如下：

\[ dp[i][j]=\sum dp[i-1][j-k] \qquad k: j - 1 \rightarrow 1 \]

为了节约空间复杂度，我们也可以使用一维数组\(dp[i]\)表示所有数组元素组成目标数字的方案数：动态转移方程如下：

\[ dp[i]=\sum dp[i-n] \qquad n \in nums \]
