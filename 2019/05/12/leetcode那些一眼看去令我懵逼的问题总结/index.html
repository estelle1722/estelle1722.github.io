LeetCode 375.Guess Number Higher or Lower II

这道题目查了很多资料，才明白这是一个minmax优化问题，简单来说，乙在1~n之间选了一个数，设选中的数字为\(x\)；甲需要猜，我们用\(y\)表示甲猜的数；每次猜完，乙会给一个反馈higher或者lower；猜错一次，都要支付\(y\)刀（可理解为代价），问题是，甲在运气最差的情况下也能猜中，最少需要付出的代价。这可以用minmax问题建模，关于minmax问题，可参看。这道题拿[1,2,3,4]为例，因为这道题关心的是运气最坏情况，因此我们只讨论最多猜多少次才能猜中的情况，过程如下：

  1. 猜1，如果一次猜对，则代价为0，否则在[2,3,4]，此时应该猜3，根据反馈一定能猜中（代价为0）；最低代价为1+3=4；
  2. 猜2，如果一次猜对，则代价为0，否则在[1]，或者[3,4]，在[1]，则直接能猜中（代价为0）；在[3,4]，我们猜3，根据反馈一定能猜中（代价为0）；最低代价为2+3=5；
  3. 猜3，如果一次猜对，则代价为0，否则在[1,2]，或者[4]，在[1,2]，猜1，根据反馈一定能猜中（代价为0），在[4]，则直接猜中（代价为0）；因此，最低代价为3+1=4；
  4. 猜4，如果一次猜对，则代价为0，否则在[1,2,3]，猜2，根据反馈一定能猜中（代价为0）；因此，最低代价4+2=6；

综上，[1,2,3,4]时，最低代价为4；

图中，边表示决策，即猜那个数字；节点表示决策最坏；只画出了在某一节点处，甲做出最优决策下但是运气最坏的结果；比如在节点[2,3,4]处，最优决策当然是3，如果运气好，当然是直接猜中；但是运气差的话，可能就直接猜错了，可能选中的数是2或者4，但是无论选中的数字是2还是4，下一次根据反馈一定能猜中。因此，在节点[2,3,4]，最优决策就是猜3；根据以上决策结果，我们可以知道，当选中的数字在[1,2,3,4]时，我们第一次猜1，第二次猜3，在猜两次的情况下，就一定能保证能猜出选中的数，并且代价最低。说到这里，这道题还可以有Follow Up：

在1~n之间选数字，最少猜多少次，一定能猜出选中的数字？

在程序实现方面，我们可以使用递归结合Cache实现，我们设dp[l][r]表示选中数字在[l,r]之间时，在满足一定能猜出选中数字条件下的最低代价。则动态转移方程为：

\[ dp[l][r] = min(i + max(dp[i+1][r], dp[l][i-1])) \qquad i:l\rightarrow r \]

外层\(min\)表示，当选中数字在[l,r]之间时，我们需要作出最优策略，在该区间内猜一个数。内层\(max\)表示，在运气最坏情况下，我们仍需要保证能猜出。例如当l=1，r=4时，我们猜2；那么我们这个决策的应该是\(max(1+dp[1][1], 1+dp[3][4])\)；显然，dp[1][1]=0，是猜2，反馈是lower情况下的最小值，也就说明选中的数字是1，这是在运气比较好的情况下；更坏的情况是，选中的数字在[3,4]；显然，题目要求一定能猜中，那么两者中的较大值（运气最坏），才能作为猜2决策下的代价。完整代码如下：
