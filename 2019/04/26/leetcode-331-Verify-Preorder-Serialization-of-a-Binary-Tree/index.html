leetcode 331.Verify Preorder Serialization of a Binary Tree

发表于 2019-04-26 | 分类于 校招笔试

这篇博客主要分析以下两个关于序列化二叉树题目：

  1. LeetCode 331.Verify Preorder Serialization of a Binary Tree
  2. 剑指Offer: 序列化二叉树

这两道题目是很类似的，都是有关于序列化二叉树的，一道来自LeetCode，一道来自剑指Offer；剑指Offer那道题涉及到两个小问题，给定二叉树，怎么将其序列化（二叉树转字符串），又如何将其反序列化（字符串转二叉树）？LeetCode那道题是给定一个字符串，判断其能否形成一个二叉树。

剑指Offer: 序列化二叉树

先来说说剑指Offer那道题目，我们以序列12##34###为例，说明其构造过程，我们都可以将序列结构看做是根节点-左子树-右子树，构造过程如下：

  1. 1为根节点，接下来字符串可分为两部分，分别代表其左子树和右子树。
  2. 2为根节点的左子树，接下来的两个##分别为其左子树和右子树，以2为根节点的子树构造完成。
  3. 3是根节点的右子树。
  4. 4是3的左子树。
  5. 接下来的两个#分别为4的左右子树，以4为根节点的子树构造完成。
  6. 最后#是3的右子树。

到此，二叉树反序列化完整，我们用createTree(ss, index)函数来表示以ss[index]为根节点的二叉树，返回值为两个，第一个为根节点指针，第二个为该二叉树最后一个节点的下一个节点（即下一次构造二叉树的起始节点），从上述构造过程可用递归函数实现。完整代码如下：

LeetCode 331.Verify Preorder Serialization of a Binary Tree

这道题，我提供两种实现：递归和栈。相对而言，栈实现效率更高，结合Python，代码可读性也更强。

递归函数

这道题还是可以沿用上面构造二叉树的思路进行，核心仍旧是函数createTree(ss, index)，初始index=0，依旧是返回两个值：

  1. 第一个值表示以ss[index]为根节点能否构造二叉树
  2. 第二个值表示下一次构造二叉树的其实节点（与上面含义一致）

不难想象，我们以index=0位置构造二叉树，如果整个序列构成一颗二叉树，则函数createTree(ss, index)第一个返回值为True，第二个返回值为ss字符串长度；否则，要么第一个返回值为False，第二个值小于ss字符串长度。在此之前，可以先处理一些边界（必要）条件（例如，二叉树的叶子节点个数等于非叶子节点加1），另外，如果左子树并不能构造为一棵二叉树，可提前结束程序，完成剪枝。最终，完整代码如下：

栈

使用栈来解决此问题的思路与递归思路稍有不同，更准确的说，更为简单一些。依次遍历序列，将字符压入到栈中，如果栈顶出现两个#，则将两个#和一个数字弹出（组成一个完整根节点-左孩子-右孩子结构），再将一个#压入栈中，遍历整个序列完成之后，判断最后栈中是否仅仅剩下一个#；以序列123####为例，遍历过程中栈中状态为：

  1. stack = [1]
  2. stack = [1,2]
  3. stack = [1,2,3]
  4. stack = [1,2,3,#]
  5. stack = [1,2,3,#,#] 将栈顶两个#和3弹出，再将#压入栈中，栈中元素为： stack = [1,2,#]
  6. stack = [1,2,#,#] 将栈顶两个#和2弹出，再将#压入栈中，栈中元素为： stack = [1,#]
  7. stack = [1,#,#] 将栈顶两个#和1弹出，再将#压入栈中，栈中元素为： stack = [#]
