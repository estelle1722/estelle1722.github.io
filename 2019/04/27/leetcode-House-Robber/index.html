LeetCode 198.House Robber题意如下：

大意是说有一个小偷晚上打算去一个社区，这个社区的房子之间是有防盗系统的，如果相邻房子被盗，则会报警，之后小偷会被抓住。这里是站在小偷角度，如果偷到最多的钱但是不触发报警系统。这道题，我最先想到的是回溯法，因为每个房子只有两中状态，偷/不偷。那么搜索空间便是有穷的，就转化为在约束条件下的最优解问题。回溯法实现代码如下：

实际上，这道题并不关心那个房子被偷了，或者说不关系最优解时每个房子是否被偷的状态。因此，回溯法这种暴力搜索的时间复杂度就显得非常高了，设给定\(n\)个元素，则上面回溯法的时间复杂度为\(O(2^n)\)。上面提到，我们并不关心具体每个房子的状态，我们可以用动态规划思路解决该题。按题意，到第\(i\)个房子时，有两个选择：偷/不偷。那么到该位置的最优解便是两种状态下的较优者。设dp[i]表示到达第\(i\)个位置时偷到的钱的最大值（偷/不偷），状态转移方程可写为：

\[ \begin{cases} dp[i] = nums[0]& \text{ if } i=0 \\ max(nums[0], nums[1]) & \text{ if } i= 1\\ max(dp[i-2] + nums[i], dp[i - 1]) & \text{ if } i\ge 2 \end{cases} \]
