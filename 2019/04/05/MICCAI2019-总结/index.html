MICCAI2019-总结

发表于 2019-04-05 | 更新于 2019-04-19 | 分类于 心得总结

MICCAI2019于4月3号结束，自己也投了两篇paper，一篇一作，一篇二作； 一作那篇是上次MIDL被拒的做biomarker discovery；另外一篇是用triplet loss做small sample分类的，说起来这篇paper还是蛮极限的，前前后后只用了五天，通宵了两个晚上，也是尽力了。赶完MICCAI2019后，突然对科研这件事有了新的体会，还是想写个总结：

  1. 科研过程中，最重要是思考，做small-sample这篇主要用的是triplet loss，在7类和198类皮肤数据集上效果都还可以，突然想到，我最近也是用triplet loss做incremental learning，一直卡在15类分类，再对比实验室另外一个同学做的project，说明triplet loss做分类还是可行的，我的project还是有问题的。
  2. 在你的project用到某一个模块或者加入一个新的loss term以解决某个问题之前，还是要达到对所使用模块非常熟悉的程度，而不仅仅是照搬过来embed到自己的project中；一旦出问题，很难准确定位到问题所在，从另一个角度来讲，就是出现一个问题，你很难对当前问题作出一个符合逻辑的解释；这一点，在科研过程中是非常重要的，对问题提出一个逻辑自洽的解释之后，接下来便可以做实验进行验证了；再举一个例子，就是在我的project中，用kNN在testing sets feature space中根据Euclidean distance计算accuracy，其实kNN可行是有一个假设条件：feature space比较致密，这也从另外一个角度说明，feature vector的维度不宜过大。
  3. 需要多读paper，以前不是很明白，也没有花很多时间，现在突然觉得，读paper非常重要，这是idea的重要来源；刚入学时，有看到一篇关于如何做科研的推文，里面写道，做科研通常步骤是先确定一个issue或者topic，接着去读相关paper，总结出目前已有的方法，接着在已有方法上提出改进；现在对这段话有了更深的理解，读paper，看别人的idea，才能进一步提出自己的idea，想来这也符合人的认知规律，人只有在有一定输入的情况下，才能可能有输出，巧妇岂能无米之炊说的也是这个道理；读paper这件事最好放在run experiments时进行。
  4. 在使用一个新的技术时，还是要从读paper开始，而不是先从code开始，这次train triplet loss便反过来了，因为先读paper有一个好处就是paper里一般会讲一些常见tricks，这是在code中急需注意到的，而且就算code是对paper的复现，所用到的数据集往往是最简单的数据集(如MNIST)，但是你自己的project中，不可能只是使用最简单的数据集；而且，对于最简单的数据集，事实往往是无任何tricks可言(怎么训练都能出比较好的结果)，这样就把一些问题给掩藏了。但是，当你用到你的project中时，这些问题多半会表现出来。其实，大部分问题，如果你事先知道tricks（如对feature做\(L_2-normalization\)），是不会出现的，这次我训练triplet loss便遇到了这种情况，耗费了大量时间，让实验继续变得异常困难。
  5. 最后，做科研这事，不要太急，慢慢来。

以上全是自己做科研一年来的体会，暂作总结经验教训。

2019年4月4日于超算。
